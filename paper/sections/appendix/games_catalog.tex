\section{Tag Assignments}
\label{app:tags}

The Kant tagging system assigns metadata across eight dimensions to each
game, enabling stratified evaluation and domain-aware splitting
(Section~\ref{sec:splitting}).

\paragraph{Tag dimensions.}
\begin{enumerate}[nosep]
\item \textbf{Cooperation pressure}: none / low / medium / high
\item \textbf{Information structure}: complete / incomplete / asymmetric
\item \textbf{Timing}: simultaneous / sequential / repeated
\item \textbf{Player count}: 2 / N / variable
\item \textbf{Fairness relevance}: low / medium / high
\item \textbf{Trust relevance}: low / medium / high
\item \textbf{Conflict type}: none / competitive / mixed-motive
\item \textbf{Mechanism design}: none / communication / voting / enforcement
\end{enumerate}

Tags enable stratified splitting: the 78/22 train/eval split guarantees
$\geq 20\%$ representation per tag value in the eval set, preventing
domain-specific overfitting.
